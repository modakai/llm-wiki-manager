# AI 个人知识库桌面端 App 功能单

## 1. 项目概述

本项目是一款支持 Windows 和 macOS 的桌面端个人知识库管理平台。

产品核心是：

- LLM 智能辅助
- AI Agent 自动整理知识
- LLM Wiki 自动生成
- Obsidian 风格笔记管理
- 多格式文件管理
- 聊天式知识库操作
- 用户手动整理知识库

其中，**LLM Wiki 不等于 RAG 知识库**。

LLM Wiki 的定位是：  
通过 LLM 和 AI Agent 将用户的笔记、文档、资料、资讯等内容自动整理成结构化的 Wiki 页面、主题目录、知识索引和知识卡片。

RAG 的定位是：  
如果后续需要实现“基于知识库聊天问答”“引用来源回答”“语义检索”等能力，可以作为可选能力接入，但不是 LLM Wiki 的必要前提。

---

## 2. 产品目标

1. 支持 Windows 和 macOS 桌面端运行。
2. 支持用户导入和管理个人知识资料。
3. 支持 doc、docx、pdf、md、txt 等主流文件格式。
4. 支持通过 AI Agent 自动整理知识内容。
5. 支持自动生成 LLM Wiki 知识库。
6. 支持类似 Obsidian 的 Markdown 笔记管理方式。
7. 支持通过聊天窗口管理笔记、资料、标签、分类和 Wiki。
8. 支持用户手动编辑、分类、整理自己的知识库。
9. 支持本地优先存储，保护用户隐私。
10. 后续可扩展语义搜索、知识问答、多媒体处理、插件系统等能力。

## 11. LLM Wiki 知识库

### 11.1 LLM Wiki 定位

LLM Wiki 是 AI 自动生成和维护的结构化知识页面系统。

它不是传统意义上的 RAG 知识库，也不是单纯的向量数据库。

LLM Wiki 更接近：

```text
原始资料
→ AI 摘要
→ 主题分类
→ 知识卡片
→ Wiki 页面
→ 主题索引
→ 关联链接
→ 可持续更新的个人知识库
```

### 11.2 Wiki 自动生成

| 功能点 | 功能说明 | 优先级 |
|---|---|---|
| Wiki 首页 | 根据知识库内容生成总览页面 | P0 |
| 主题页 | 按主题生成独立 Wiki 页面 | P0 |
| 目录结构 | 自动生成多级知识目录 | P0 |
| 概念解释 | 提取核心概念并生成解释 | P0 |
| 关联链接 | 为相关概念和笔记建立链接 | P1 |
| 学习路径 | 根据知识点关系生成学习路径 | P1 |
| FAQ | 从文档中提取常见问题 | P1 |
| 术语表 | 生成术语词典 | P1 |

### 11.3 Wiki 页面结构

每个 Wiki 页面建议包含：

| 区域 | 说明 |
|---|---|
| 页面标题 | 当前知识主题名称 |
| 简介 | 主题概述 |
| 核心概念 | 该主题下的关键知识点 |
| 相关笔记 | 关联的原始笔记 |
| 相关文件 | 关联的 PDF、doc、md、txt 等文件 |
| 知识卡片 | AI 提取的结构化知识点 |
| 引用来源 | 内容来源文件和段落 |
| 相关主题 | 与其他 Wiki 页面建立链接 |
| 更新时间 | 页面最近生成或修改时间 |

## 24. 核心业务流程

### 24.1 文件导入到 LLM Wiki 生成流程

1. 用户拖拽 PDF、docx、md、txt 文件进入客户端。
2. 系统解析文件内容。
3. 系统进行文本清洗。
4. AI Agent 提取标题、摘要、关键词、标签。
5. AI Agent 判断文件所属分类。
6. AI Agent 推荐 Wiki 主题。
7. AI Agent 生成对应 Wiki 页面或更新已有页面。
8. 用户确认整理结果。
9. 系统写入知识库。
10. 系统记录修改日志，支持回滚。

### 24.2 聊天管理知识库流程

1. 用户打开聊天窗口。
2. 用户输入：“帮我整理最近导入的 AI Agent 资料。”
3. 系统查找最近导入内容。
4. Agent 分析文件内容。
5. Agent 给出整理方案。
6. 用户确认。
7. 系统执行分类、打标签、生成 Wiki、建立关联。
8. 系统展示执行结果和修改记录。

### 24.3 手动整理流程

1. 用户进入文件树。
2. 用户手动新建目录。
3. 用户拖拽文件到目录中。
4. 用户打开笔记进行编辑。
5. 用户添加标签、双链、附件。
6. 保存后系统更新文件元数据。
7. 相关 Wiki 页面提示可更新。

## 25. 非功能性需求

| 类型 | 要求 |
|---|---|
| 性能 | 普通知识库搜索响应尽量控制在 1 秒内 |

| 隐私 | 默认本地存储，外部模型调用需要明确提示 |
| 可扩展性 | 模型、文件解析器、导出格式应可扩展 |
| 兼容性 | 兼容 Windows/macOS 文件路径、权限、编码差异 |
| 可恢复性 | AI 修改必须支持查看记录和回滚 |
| 易用性 | 用户可以不懂技术，通过聊天完成知识整理 |
| 可维护性 | 数据结构清晰，便于后续同步和导出 |
| 安全性 | API Key、本地知识库支持加密保存 |

## 27. 产品一句话总结

这是一款面向个人用户的 AI 原生知识库桌面应用，将 Obsidian 式 Markdown 笔记管理、LLM Wiki 自动生成、AI Agent 知识整理、多格式文件管理和聊天式操作结合在一起，让用户可以通过 AI 自动管理和持续沉淀自己的知识资产。
